# DAY14-NOTE

## 深度学习

### 过拟合

1.定义：过拟合是指模型在训练数据上表现极好（如损失趋近于0），但在未见过的测试数据上表现很差的现象

2.为什么会发生过拟合

【1】模型复杂度过高：如神经网络层数太多、参数太多，导致模型容量过大

【2】训练数据不足：样本太少，无法覆盖真实分布

【3】数据噪声过大：训练集中存在错误标签或异常值，模型强行拟合这些噪声

【4】训练时间过长：模型在训练集上反复迭代，逐渐记住细节而非泛化特征

### 归一化

1.定义：数据归一化是将数据按比例缩放，使之落入一个小的特定区间（如[0,1]或[-1,1]），或使其均值为0、标准差为1的过程

2.常见归一化方法：

【1】Min-Max 归一化 

【2】 Z-Score 标准化

【3】 Robust Scaling

### 正则化

1.定义：为了防止过拟合而对模型施加的约束或惩罚机制，通过限制模型复杂度，提升泛化能力。（泛化 = 模型在新数据上也能用好的能力）

2.常见的正则化方法:

【1】Lasso：在损失函数中加入权重的绝对值之和

【2】Ridge：加入权重的平方和

【3】Elastic Net：加入【1】【2】之和

【4】Dropout（神经网络特有）：训练时，随机关闭一部分神经元

【5】早停法：当验证集误差开始上升时停止训练

【6】数据增强：通过对训练数据做变换（如旋转、裁剪、加噪声）

### 数据清洗

1.定义：数据清洗是发现并纠正数据中错误、不完整、不一致或无关部分的过程，是机器学习中最耗时但最重要的环节

### Batch Size

1.定义：Batch Size 是指一次前向+后向传播中使用的样本数量

### 学习率

1.定义：学习率控制参数更新的步长：w=w−η⋅∇L，η是学习率
